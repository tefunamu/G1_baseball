# ⚾ G2 BASEBALL — Even G2 タイミング野球ゲー 設計図

最終更新：2026-06-30 ／ コードネーム "G2 BASEBALL"

## 🎯 ゴール
ロボ(Unitree G1)が投げる → 人がEven G2で**迫ってくるボール**を見る →
タイミングよくバットを振る → **HIT/空振り判定** → 結果演出（飛距離・実況）。

※真の空間AR（色・3D・当たり判定）はG2では不可（緑単色HUD・カメラ/空間認識なし）。
　なので**「タイミング野球ゲー」に割り切る**＝G2の身の丈で"体験"を成立させる。

---

## 🧩 全体構成（4部品）
1. **ピッチャー＝Unitree G1**：投球モーション＋「今投げた！」イベントを発火
   （メガネがCVで認識…ではなく、**ロボが自分で合図する**のがミソ＝CV不要で確実）。
2. **ゲームサーバー**（Mac mini・新規・eventermの兄弟プロセス）：
   投球イベント受信 → アニメの時間割を組む → G2へフレーム送出 → スイング受信 → 判定 → 結果送出。
3. **表示＝Even G2**：既存の eventerm `/api/push`（実装済）に**テキストフレーム**を投げて表示。
   本格演出は Even Hub プラグインで丸を実描画（後段）。
4. **スイング検出＝バットに付ける M5StickC**：加速度しきい値でスイングを検出→`/swing` をPOST。
   ※あなたのM5フラッシュ環境（Stack-chan由来）がそのまま活きる。代替＝スマホIMU／物理ボタン。

---

## 🔁 データの流れ
```
G1 [throw event] ──▶ Server   (t0, 到達まで flight=約1.6s)
Server ──▶ G2 : フレーム送出（遠→中→近→「打て!」）
M5(bat) [swing event, t_swing] ──▶ Server
Server : |t_swing − t_arrival| を判定（±窓）→ Perfect / Good / Late / 空振り
Server ──▶ G2 : 結果（HIT! ホームラン! / ストライク…）
Server ──▶ VOICEVOX+Stack-chan : 音声実況＋首振り
```

---

## ⏱ タイミング設計（重要・公平性）
- BT表示もセンサーも**遅延がある**ので、**ミリ秒精密は狙わない**。
- 投球を**ゆっくり（1.5〜2秒の山なり）**にして**判定窓を広め**に＝まず「当たる楽しさ」優先。
- **キャリブレーション**：1球テスト投げ→「表示遅延＋スイング遅延」の合計オフセットを実測→判定窓を補正。

## 🪜 段階ビルド（クロール→ウォーク→ラン）
- **P0 サーバーだけ**：タイマー投球→コンソールで判定ロジック検証（ロボ/メガネ不要・今すぐ）
- **P1 表示**：`/api/push` でG2に「迫るボール」フレームを出す（既存配管＝最速の手応え）
- **P2 入力**：M5StickCをバットに→スイング検出→`/swing`→**ソロでプレイ可**（サーバーがタイマー投球）
- **P3 ロボ連動**：G1 SDKが投球イベント発火→サーバー＝**本来の絵が完成**
- **P4 仕上げ**：得点・実況音声(VOICEVOX/Stack-chan)・球速バリエ・連続演出
