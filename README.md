# ⚾ G1_baseball — Even G2 タイミング野球ゲー

ロボット(Unitree G1)が投げ、人が **Even G2スマートグラス** で迫ってくるボールを見て、
タイミングよくバットを振って打ち返す——**タイミング野球ゲー**。

> Even G2 は緑単色・低解像度の **HUD（情報表示メガネ）** で、空間AR（色つき3D・当たり判定）は不可。
> なので「ボールが迫る→打つタイミングを当てる」体験に割り切った設計。
> 真の空間ARが必要なら Meta Quest 3 等が別途必要。

---

## 🏗 構成（3部品 + 2つの外部プラグ）

```
[G1ロボ] --POST /pitch--> [ゲームサーバー(本リポ)] --表示--> [Even G2]
[スマホ] --POST /swing--> [ゲームサーバー]  --判定/スコア--> [Even G2]
```

- **ゲームサーバー（このリポジトリ）** = `server.mjs`。投球受付・タイミング判定・スコア・G2表示。
- **ピッチ入力** = G1ロボが `POST /pitch` を叩く（当日）。開発中はブラウザのボタン/タイマーで代用。
- **スイング入力** = スマホ等の加速度検出が `POST /swing` を叩く。開発中はブラウザのボタンで代用。
- **表示** = 既存の [even-terminal 互換サーバー(eventerm)](https://github.com/) の `POST /api/push` 経由で
  Even G2 にテキストフレームを送る（`遠 → 中 → 近 → 打て! → 結果`）。

外部2パーツ（スイング検出・ロボ）の**合流仕様は [INTEGRATION.md](./INTEGRATION.md)** 参照。
設計の詳細は [DESIGN.md](./DESIGN.md)。

## 🚀 使い方

### 1. 必要なもの
- [Bun](https://bun.sh/)（`server.mjs` を起動するため。Node.js でも可）
- （メガネ表示する場合のみ）eventerm サーバーが `localhost:3456` で起動していること＋
  `~/even-g2/even-terminal-bridge/token.txt` にトークンがあること。
  → 無くても**ゲームのロジックとブラウザ操作は動く**（メガネ表示だけスキップ）。

### 2. 起動
```bash
bun run server.mjs
# → ⚾ G2 BASEBALL server on :3457
```

### 3. 遊ぶ（ブラウザのテスト台）
ブラウザで **`http://localhost:3457/`**（別端末からは `http://<サーバーIP>:3457/`）を開く。
- 「遅球 / 普通 / 速球」= 投球
- 「🏏 SWING」= スイング
- 下にスコアが出る。指でタイミングゲーが遊べる。

### 4. Even G2 に表示する
メガネ側の **Terminal モードで会話セッションを開いた状態**にしておくと、
ボールのフレームがメガネに流れる。
（※セッションを開いていない＝待機/一覧画面だと、表示は届かない。ロジックは動く。）

## 📡 エンドポイント

| Method | Path | 用途 |
|---|---|---|
| `POST` | `/pitch` | 投球開始。body `{ "speed": "slow"\|"normal"\|"fast" }`（省略時 normal） |
| `POST` | `/swing` | スイング登録。body 空でOK（任意で `{ "t": <epoch_ms> }`） |
| `GET`  | `/state` | 現在の状態とスコア `{ active, last, score }` |
| `GET`  | `/health`| 死活確認 `{ ok: true }` |
| `POST` | `/reset` | スコア初期化 |
| `GET`  | `/` | ブラウザ用テスト台 |

### 判定ルール
到達時刻との差 `diff`（- が早い / + が遅い）の絶対値で判定:

| 条件 | 結果 |
|---|---|
| ±150ms 以内 | **ホームラン** |
| ±350ms 以内 | ヒット |
| ±600ms 以内 | ファウル |
| それ以上 | 空振り |
| スイング無しで時間切れ | 見逃しストライク |

球速→到達時間: slow=2.0s / normal=1.6s / fast=1.2s

## ⏱ タイミングについて（重要）
Bluetooth 表示も通信も遅延があるため、**ミリ秒精密は狙わない**設計。
球を山なり（1.5〜2秒）にして判定窓を広めに取り、「当てる楽しさ」を優先。
本番前に1球テストして遅延オフセットを実測し、判定窓を調整すること。
